# Manual Numbering Policy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an opt-in `Layout.Numbering = "manual"` mode that removes DOCX automatic paragraph numbering so callers can write numbering text themselves.

**Architecture:** Keep automatic numbering unchanged by default. Add a root-level `NumberingPolicy` enum on `Layout`; when set to `manual`, `Apply` strips paragraph `w:numPr` from rebuilt blocks after edits and before write-back. Implement the XML mutation in `internal/ooxml` as a small helper that removes direct paragraph numbering properties without touching other paragraph properties.

**Tech Stack:** Go 1.25, standard library XML/token scanning already used in `internal/ooxml`, existing DocxTidy root API.

## Task 2: Remove Paragraph Numbering in Manual Mode

- [ ] **Step 1: Add internal helper**

In `internal/ooxml/numbering.go`, add:

```go
func RemoveParagraphNumbering(blockXML string) string {
	start := strings.Index(blockXML, "<w:numPr>")
	if start == -1 {
		return blockXML
	}
	end := strings.Index(blockXML[start:], "</w:numPr>")
	if end == -1 {
		return blockXML
	}
	end += start + len("</w:numPr>")
	return blockXML[:start] + blockXML[end:]
}
```

This handles the direct `w:numPr` generated by current fixtures and the target WPS/Word paragraphs.

- [ ] **Step 2: Apply helper in manual mode**
